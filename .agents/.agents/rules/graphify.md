# Graphify Workflow Rule

**ALWAYS FOLLOW THIS WORKFLOW:**

1. **Query First**: Before scanning the codebase for answers or architectural context, you MUST scan the `graphify-out` directory and use the `graphify-windows` skill to query the knowledge graph. This improves token utilization and provides a clear map of the project.
2. **Keep the Graph Updated**: Whenever you create new pages, components, delete files, or make significant structural/architectural changes, you MUST add/remove them from the knowledge graph so that it remains the single source of truth.
